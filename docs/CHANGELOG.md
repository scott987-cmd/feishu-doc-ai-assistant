# 更新日志

本项目遵循[语义化版本](https://semver.org/lang/zh-CN/)。

## [Unreleased]

### 新增
- **网页剪藏 → 飞书 Base（Web Clipper, v1）**：在任意网页**右键 / 点扩展图标 / 快捷键(Alt+Shift+C)**，
  把**选中内容或整页正文**经 AI 整理成记录写入指定 Base 表。
  - **安全设计**：仅在用户**手势触发**时通过 `chrome.scripting.executeScript` + `activeTab`
    读取**当前页** DOM —— **不新增 host_permissions、不新增出站端点**（读 DOM 是本地行为；数据仍只发往
    LLM + 飞书）。发送前**预览**抓取内容（知情同意）；剥离 `<input>`/表单/脚本/页面 chrome，输入框里的
    密码等永不被抓取；写入仍走既有用户身份与安全卡点。
  - 构建开关 `VITE_CLIP_ENABLED`（默认开）；企业域名白名单 `VITE_CLIP_MANAGED_DOMAINS`（v2 才强制）。
- **网页剪藏增强**：多目标（多维表格 / 电子表格 / 文档 + 新建）；多策略表格抓取（原生 / ARIA / ag-Grid 按
  col-id 对齐 / 无语义 div 网格）；**截图 + 视觉模型**识别 canvas/图片表格；**全量抓取**（滚动加载虚拟网格全部行）；
  **拖拽 CSV/TSV/文本文件**导入；**采集模板**（按网站记住目标，一键重复）；wiki 链接解析；模型兜底重构（非表格也整理成表）。
- **AI 数据可视化 / Data App** ✨：一句话/对话把当前表做成 **ECharts 看板**，渲染成飞书页面上的悬浮窗
  （可拖拽 + 四角/四边缩放）。**代码与数据分离**——保存后用最新数据零 LLM 一键重开；一页可挂多个独立看板（各自浮标）；
  单看板可含多图。生成代码跑在 **MV3 沙箱**（null 源、`connect-src 'none'`、无 chrome.*）→ 拿了数据也发不出去。
  仅新增 echarts 依赖（隔离进沙箱包）。
- **场景 tab 改为功能中心**：模版库收成入口卡片（单独页面），新增能力以卡片展示。

### 修复
- **OAuth 自动续期补反应式重试**：原仅"过期前 5 分钟"主动续期，窗口漏掉就硬 401。新增 token 失效检测
  + `forceRefreshUserToken` 强制续期 + 工具层重试一次；refresh_token 也失效则提示重新授权。
- 剪藏/可视化一连串体验修复（链接点击、面板崩溃、最近用过下拉、Function 语法兼容、调整功能、多图重叠等）。

## [1.0.4] - 2026-06-01

### 修复
- **LLM API Key 在切换页面后丢失**：`crypto.ts` 的设备种子存在并发竞态——首次保存时
  `Promise.all` 并发加密 token 与 apiKey，两个调用各自生成不同的设备种子，只有最后写入的那个被持久化，
  导致用另一个种子加密的字段在重新加载时无法解密。现将种子与密钥派生收敛为共享缓存 Promise，
  彻底消除竞态。
- **侧边栏面板在站内导航时被反复重挂载**：`background` 此前在每次 tab 事件都重设 `setOptions({ path })`，
  会重载已打开的侧边栏。改为按 tab 缓存启用状态，仅在「飞书↔非飞书」切换时才动面板。
- **新建 Base/文档后的链接点不开 / 显示为纯文本 URL**：
  - 侧边栏里 `<a target="_blank">` 的点击常被吞掉——改为统一经 `chrome.tabs.create` 打开新标签页
    （新增共享工具 `openUrlInNewTab`，聊天链接与场景「创建完成」卡片均接入）。
  - 模型常把 URL 包进反引号/代码块导致渲染成不可点击的等宽纯文本——渲染层现会把「内容就是 URL」的
    内联代码、代码块、以及被反引号包住的 Markdown 链接照样渲染为可点击链接；提示词同步禁止把打开链接
    放进代码格式。

### 新增
- **Markdown 表格渲染**：助手回复里的 GitHub 风格表格（`| … |` + `|---|` 分隔行）现渲染为真正的表格，
  而非一堆原始竖线。
- **设置页版本号显示**：「设置」左下角显示当前运行的扩展版本（`v1.0.4`），方便确认是否已加载新构建。

### 测试
- 新增针对设备种子并发竞态、代码包裹 URL 的链接化、Markdown 表格、链接点击经 `chrome.tabs.create`
  的回归测试。当前 **193 passed | 32 skipped**。
