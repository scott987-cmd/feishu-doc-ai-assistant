🧰 企业打包向导

几步点出一个定制扩展包：改名称 / 换图标 / 配参数 → 一键打包下载。无需命令行。

① 打包模式

先选你要的场景，下面只显示对应需要填的项。

🏢 企业 / 代理（推荐）给公司用。密钥放服务端代理，员工免配置；可统一下发 App ID / 大模型 / 策略。
👤 个人 / 直连自己用。填你自己的飞书 App ID + Secret（加密存本机，不进包也可）。
🛒 商店上架（BYO）上架 Chrome 商店。不内置任何凭据、用户自填、剥掉 key。
🔒 私有化 / 内网Lark / 飞书私有化部署，填你的域名后缀。

② 品牌定制

改成你公司的名字和图标（写进扩展 manifest）。

③ 连接配置

App ID 也从代理下发勾选则无需在包里内置 App ID（单一构建服务任意租户）。不勾选请在下方填 App ID。
大模型也从代理下发员工免填 API Key（密钥在服务端）。
锁定为企业统一大模型禁止员工切到“手动配置”。
更安全：用密码加密的密文（高级）
商店模式不内置任何凭据（用户首配时自填 App ID/Secret），并自动剥掉 manifest key。无需在此填连接。

④ 企业能力（可选）

需要服务端对应子服务支持（详见文档）。

统一策略下发强制并锁定 自动确认/越用越聪明 等开关。
共享技能库多用户脱敏经验汇聚、主动推送。
企业云备份小程序/建站/PPT 镜像到企业自有对象存储。

⑤ 高级（一般不用动）

展开高级参数
外发前脱敏发给大模型前掩盖手机/邮箱/身份证。
无远程代码看板/建站走声明式渲染，沙箱去掉 unsafe-eval。
托管大模型 key 仅内存不落盘，每会话重取。
去掉内置 key首次自分发会生成新扩展 ID（商店模式自动剥）。
